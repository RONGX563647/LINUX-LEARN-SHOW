## 2.2 Linux下C程序的编写与gcc编译

### 2.2.1 Hello gcc程序

#### 程序代码

创建文件 `hello.c`：

```c
#include <stdio.h>

int main() {
    printf("Hello gcc!\n");
    return 0;
}
```

#### gcc编译过程详解

**① 预处理阶段（-E选项）**

```bash
gcc -E hello.c -o hello.i
```

**说明：** 预处理阶段处理以`#`开头的预处理指令，包括头文件包含、宏定义展开、条件编译等。生成的`.i`文件是经过预处理的源代码。

**② 编译阶段（-S选项）**

```bash
gcc -S hello.i -o hello.s
```

**说明：** 编译阶段将预处理后的代码转换为汇编代码。生成的`.s`文件包含汇编语言代码。

**③ 汇编阶段（-c选项）**

```bash
gcc -c hello.s -o hello.o
```

**说明：** 汇编阶段将汇编代码转换为目标文件（机器码）。生成的`.o`文件是二进制目标文件，还不能直接执行。

**④ 链接阶段（-o选项）**

```bash
gcc hello.o -o hello
```

**说明：** 链接阶段将目标文件与库文件链接，生成可执行文件。生成的`hello`是最终的可执行程序。

**⑤ 一步编译**

```bash
gcc hello.c -o hello
```

**说明：** 可以直接从源文件生成可执行文件，gcc会自动完成所有阶段。

**⑥ 调试版本编译（-g选项）**

```bash
gcc -g hello.c -o hello_debug
```

**说明：** `-g`选项在可执行文件中包含调试信息，可以使用gdb等调试工具进行调试。

#### 运行程序

```bash
./hello
```

输出结果：`Hello gcc!`，程序运行成功，结果符合预期。

### 2.2.2 多文件程序与Makefile

#### 程序代码

**① 头文件 greeting.h**

```c
#ifndef GREETING_H
#define GREETING_H

void print_greeting(const char *name);
int add(int a, int b);

#endif
```

**② 自定义函数文件 greeting.c**

```c
#include <stdio.h>
#include "greeting.h"

void print_greeting(const char *name) {
    printf("Hello, %s! Welcome to Linux programming.\n", name);
}

int add(int a, int b) {
    return a + b;
}
```

**③ 主函数文件 myapp.c**

```c
#include <stdio.h>
#include "greeting.h"

int main() {
    print_greeting("Student");
    
    int result = add(10, 20);
    printf("10 + 20 = %d\n", result);
    
    return 0;
}
```

#### Makefile文件

创建文件 `Makefile`：

```makefile
# 编译器设置
CC = gcc
CFLAGS = -Wall -g

# 目标文件
TARGET = myapp

# 源文件
SRCS = myapp.c greeting.c
OBJS = $(SRCS:.c=.o)

# 默认目标
all: $(TARGET)

# 链接生成可执行文件
$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) -o $@ $^

# 编译源文件
%.o: %.c greeting.h
	$(CC) $(CFLAGS) -c $<

# 清理目标文件
clean:
	rm -f $(OBJS) $(TARGET)

# 重新编译
rebuild: clean all

# 伪目标
.PHONY: all clean rebuild
```

#### 编译和运行

**① 使用make编译**

```bash
make
```

输出：
```
gcc -Wall -g -c myapp.c
gcc -Wall -g -c greeting.c
gcc -Wall -g -o myapp myapp.o greeting.o
```

**② 运行程序**

```bash
./myapp
```

输出结果：
```
Hello, Student! Welcome to Linux programming.
10 + 20 = 20
```

程序运行成功，结果符合预期。

**③ 清理目标文件**

```bash
make clean
```

**④ 重新编译**

```bash
make rebuild
```

#### Makefile说明

1. **变量定义：**
   - `CC`：指定编译器为gcc
   - `CFLAGS`：编译选项，`-Wall`显示所有警告，`-g`包含调试信息
   - `TARGET`：目标可执行文件名
   - `SRCS`：源文件列表
   - `OBJS`：目标文件列表

2. **依赖关系：**
   - `myapp`依赖于`myapp.o`和`greeting.o`
   - `myapp.o`依赖于`myapp.c`和`greeting.h`
   - `greeting.o`依赖于`greeting.c`和`greeting.h`

3. **自动变量：**
   - `$@`：目标文件名
   - `$^`：所有依赖文件
   - `$<`：第一个依赖文件

4. **模式规则：**
   - `%.o: %.c`：通用的编译规则，将所有`.c`文件编译为`.o`文件
